# What Is Phantom Grid?

---

## The short version

Phantom Grid is a record label. It exists as code.

Not a label with a website. Not a label that uses apps and tools. A label whose entire brain — the rules, the decisions, the aesthetic system, the release process — lives in a public code repository. Version-controlled. Permanent. Open for anyone to read.

The team that operates this infrastructure is artificial intelligence. The founder is human. The label is not located anywhere, it`s a part of cyberspace.

---

## For people who don't know what a code repository is

Most record labels work like this: people in a room make decisions. Those decisions live in their heads, in emails, in conversations. When those people leave, the knowledge leaves with them.

Phantom Grid works differently.

Every decision the label has ever made is written down — not in a document that sits in a drawer, but in a living system that a computer can read and act on. What the label sounds like. How a release gets made. Who does what. What the visual language is. All of it. Written. Committed. Saved permanently.

Even if something gets changed or deleted, the original is still there — preserved in the history of the system. The label never forgets anything it has ever decided.

The team that follows these rules is not a group of people. It is artificial intelligence — three specialized agents, each with a defined role: one handles art direction, one handles A&R and music curation, one handles communication. They work from the written system. The founder, Void, makes the final calls.

The label has no physical address. It exists in Cyberspace. It can be accessed from anywhere because it lives nowhere.

## For music industry people

Phantom Grid is the first record label in the world whose entire operational infrastructure is a public, version-controlled code repository.

A&R decisions, brand guidelines, release protocols, inter-role communication — all of it is documented as code. Not as PDFs or brand decks. As a structured file system that is readable by both humans and machines.

The label's team consists of AI personas with specific mandates — not general-purpose assistants, but agents built around defined roles with aesthetic positions and documented workflows. The system was designed so that loading a role file into an AI session produces consistent, label-appropriate outputs every time.

Every release is tracked in the repository. Every version of every document is preserved. The git history is the label's institutional memory — complete and permanent.

The sound spans Electro, Techno, Minimal, and Ambient. Detroit was the spark. What has been built from it is something new.

## For developers and tech people

Phantom Grid is a record label whose operational layer is a public GitHub repository. The OS defines roles (system prompts for Claude agents), brand system, sonic brief, release protocols, and inter-agent handoff workflows.

The stack:
- **OS:** this repo — markdown files, version-controlled, MIT licensed
- **Agents:** role files as system prompts, dispatched via AI SDKs
- **Pipeline:** in development — release commit triggers Bandcamp upload, social posts, RA submission
- **Website:** Astro — separate repo, consumes OS data

The repository is open source. Anyone can fork it and build their own label OS. The fork carries the structure. It does not carry the history, the formation, or the transmission that produced the original.

A fork is not the original. Structure is copyable. Voice is not.
